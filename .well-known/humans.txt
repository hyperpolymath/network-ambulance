/* PHILOSOPHY */

"The network should work. When it doesn't, fix it quickly and safely."

This tool embodies:
1. Simplicity: Use standard tools, no exotic dependencies
2. Safety: Backup before changing, dry-run to preview
3. Transparency: Log everything, explain what's happening
4. Accessibility: Beginner-friendly AND expert-powerful
5. Reliability: Work offline, handle edge cases gracefully
6. Community: Open source, welcoming contributors
7. Security: Sanitize inputs, check privileges, fail safely
8. Documentation: README-driven development
9. Testing: Test what you build, build what you test
10. Ethics: Respectful, inclusive, emotionally safe
